# 轻量图片素材库

这是一个基于 Android Jetpack Compose 实现的双列图片流应用。项目最初来源于“参考抖音经验栏，实现双列图片列表和图文详情页”的课后作业要求，后续在功能和结构上逐步扩展成一个轻量级图片素材库。

应用的核心目标不是简单展示几张网络图片，而是让用户可以发现素材、收藏素材、找回最近看过的素材，并把需要的高清图片保存到系统相册中。

## 主要功能

- 双列瀑布流图片列表，视觉上接近素材发现流。
- 使用 OkHttp 请求网络图片。
- 首页支持“发现 / 收藏 / 最近”三个板块。
- 发现页支持下拉刷新和触底分页加载，每次追加一批素材。
- 支持搜索素材用途、场景关键词和图片分析标签，例如 `PPT`、`壁纸`、`封面`、`暖色`、`横图`。
- 列表图片按可见区域优先加载，减少一次性加载大量图片带来的等待。
- 图片加载时使用骨架屏占位，避免纯转圈造成明显等待感。
- 点击图片进入详情页，展示图片、描述、尺寸、大小、缓存位置、色彩分析等信息。
- 详情页支持高清图加载、全屏大图预览、缩放和拖动。
- 支持收藏图片，收藏内容持久化保存。
- 支持最近看过，只记录用户真正点进详情的图片。
- 支持保存高清图片到系统相册。
- 支持内存 LRU 缓存、磁盘缓存、临时文件下载校验和缓存大小控制。

## 技术实现

项目保留了初学者手写网络、缓存和图片处理逻辑的学习过程，没有直接使用 Glide 或 Coil 接管图片加载。主要技术点包括：

- Kotlin + Jetpack Compose 构建界面。
- ViewModel + UiState 管理页面状态。
- Repository 统一协调网络、缓存、收藏、最近和保存相册逻辑。
- OkHttp 负责网络图片下载，并配置超时、重试和并发控制。
- BitmapFactory 负责本地图片解码，并通过采样解码降低内存压力。
- LruCache 保存内存 Bitmap，减少列表反复滑动时的重复解码。
- 文件缓存保存缩略图和高清图，避免已下载图片重复请求。
- 下载时先写入 `.download` 临时文件，确认可解码后再替换为正式缓存文件。
- 图片加载调度区分当前可见、附近预加载和远处预取，尽量兼顾滑动速度和加载稳定性。

## 项目结构

```text
app/src/main/java/com/example/image
├── MainActivity.kt
├── data
│   ├── cache          # 内存缓存、磁盘缓存、采样解码、缓存清理
│   ├── model          # 图片模型、加载状态、筛选类型、加载优先级
│   ├── remote         # OkHttp 网络请求
│   └── repository     # 图片数据、收藏、最近、保存相册等业务逻辑
├── ui
│   ├── detail         # 图片详情页和大图预览
│   ├── home           # 首页、搜索、瀑布流、图片卡片
│   ├── theme          # Compose 主题
│   └── viewmodel      # GalleryViewModel 和 GalleryUiState
└── util               # 图片分析、格式化、状态文案
```

## 运行方式

使用 Android Studio 打开项目，等待 Gradle 同步完成后运行 `app` 即可。

也可以在项目根目录执行：

```bash
./gradlew :app:assembleDebug
```

Windows PowerShell 下可以执行：

```powershell
.\gradlew.bat :app:assembleDebug
```

生成的 Debug APK 位于：

```text
app/build/outputs/apk/debug/app-debug.apk
```

## 权限说明

项目使用以下权限：

- `INTERNET`：用于请求网络图片。
- `WRITE_EXTERNAL_STORAGE`：仅用于 Android 9 及以下系统保存图片到相册。

Android 10 及以上系统主要通过 MediaStore 保存图片，不需要传统外部存储写入权限。

## 设计思路
项目经历了几个阶段：

1. 先完成作业要求：网络图片、双列列表、详情页和缓存。
2. 再从用户角度重新定位：把普通图片列表调整为“轻量图片素材库”。
3. 然后补充真实使用路径：收藏、最近、搜索、保存相册、分页加载和大图预览。
4. 最后整理工程结构：拆分 data、remote、cache、repository、ui、util，避免所有逻辑堆在 Activity 中。
